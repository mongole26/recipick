# New User Onboarding

## 목표

첫 진입부터 레시피 추천까지 **1분 이내**에 도달하게 한다.
회원가입 강제 없이, 재료 입력의 재미를 먼저 경험시킨다.

---

## 온보딩 플로우

```
앱 최초 실행
  ↓
[스플래시] — 로고 + 1초
  ↓
[온보딩 카드 3장] — 스와이프
  ↓
[홈 화면] — 바로 사용 가능 (비회원)
```

---

## 온보딩 카드 구성

| 순서 | 내용 | 비주얼 |
|------|------|--------|
| 1 | "냉장고에 뭐가 있나요?" | 냉장고 일러스트 + 재료 아이콘 |
| 2 | "재료를 입력하면 레시피를 찾아드려요" | 검색창 → 레시피 카드 전환 애니메이션 |
| 3 | "지금 바로 시작해 보세요" | CTA 버튼: "시작하기" |

### 디자인 원칙
- 텍스트는 한 문장씩만
- Skip 버튼 항상 노출 (우상단)
- 페이지 인디케이터 (점 3개)
- 마지막 카드에서 "시작하기" 버튼 → 홈 화면 이동

---

## 첫 사용 유도 (홈 화면)

비회원으로 진입한 직후, 냉장고가 비어있는 상태:

```
┌──────────────────────────────────┐
│                                   │
│         🧊 (냉장고 아이콘)         │
│                                   │
│     냉장고가 비어있어요             │
│     재료를 추가해 보세요            │
│                                   │
│     [ + 재료 추가하기 ]            │  ← 메인 CTA
│                                   │
└──────────────────────────────────┘
```

- 빈 상태(empty state)가 곧 CTA
- 재료 1개 이상 추가되면 "레시피 추천받기" 버튼 활성화

---

## 로그인 유도 타이밍

강제하지 않고, 자연스러운 시점에 노출:

| 시점 | 메시지 |
|------|--------|
| 추천 결과 확인 후 | "로그인하면 냉장고 데이터가 저장됩니다" (하단 배너) |
| 재방문 시 (로컬 데이터 있을 때) | "로그인하면 다른 기기에서도 사용할 수 있어요" |
| 즐겨찾기 시도 시 (P1) | "로그인 후 저장할 수 있어요" (바텀시트) |

### 원칙
- 첫 추천까지는 로그인 유도 **없음**
- 팝업 아닌 인라인 배너로 노출
- "나중에" 선택 시 다시 안 보임 (세션 단위)

---

## 재진입 시나리오

| 상태 | 동작 |
|------|------|
| 비회원 + 냉장고 있음 | 홈 화면에 재료 현황 표시, "추천받기" 바로 가능 |
| 비회원 + 냉장고 없음 | 빈 상태 + 재료 추가 CTA |
| 로그인 회원 | Supabase에서 냉장고 데이터 동기화 후 표시 |
