# Recipick Product Specs

## 한 줄 정의

냉장고에 남은 재료를 입력하면, AI가 만들 수 있는 레시피를 추천해주는 앱.

---

## 타겟 사용자

| 세그먼트 | 특징 | 핵심 니즈 |
|----------|------|-----------|
| **자취생 / 1인 가구** | 요리 경험 적음, 간단한 재료 보유 | "이것만으로 뭘 해먹지?" |
| **일반 가정** | 다양한 재료 보유, 반복 메뉴 고민 | "매일 뭘 해먹지? 색다른 거 없을까?" |

공통 페인포인트: **"냉장고에 뭐가 있는데, 이걸로 뭘 만들 수 있는지 모르겠다."**

---

## 핵심 가치

1. **재료 기반 역발상** — 레시피를 찾고 장을 보는 게 아니라, 있는 재료로 시작
2. **진입 장벽 제로** — 로그인 없이 바로 사용, 재료 입력이 직관적
3. **신뢰할 수 있는 추천** — 매칭률 + AI 설명으로 "왜 이 레시피인지" 납득 가능

---

## 톤 & 브랜드 가이드

| 항목 | 기준 |
|------|------|
| **전체 톤** | 실용적, 깔끔, 군더더기 없음 (컬리 레퍼런스) |
| **텍스트** | 짧고 명확한 문장, 존댓말, 불필요한 이모지 자제 |
| **UI** | 화이트 베이스, 충분한 여백, 카드 기반 레이아웃 |
| **색상** | 메인 #5F0080 (보라), 보조 #F3E8FF, 배경 #F7F7F7 |
| **AI 응답** | 친절하지만 간결, 추천 이유를 한 문장으로 |

## MVP 기능 목록

### P0 (필수)

| 기능 | 설명 | 상태 |
|------|------|------|
| 재료 입력 | 검색 자동완성 + 수량/단위 선택 (바텀시트) | ✅ 완료 |
| 냉장고 관리 | 등록된 재료 리스트, 삭제 | ✅ 완료 |
| 레시피 추천 | 재료 기반 3개 추천 (RAG 파이프라인) | 🔲 |
| 추천 결과 화면 | 카드형 리스트 (아래 상세) | 🔲 |
| 레시피 상세 | 재료, 조리 순서, 팁 | 🔲 |

### P1 (MVP 직후)

| 기능 | 설명 |
|------|------|
| 소셜 로그인 | Google, 카카오 |
| 냉장고 동기화 | 로컬 → 클라우드 마이그레이션 |
| 즐겨찾기 | 레시피 저장 |

### P2 (이후)

| 기능 | 설명 |
|------|------|
| 조리 기록 | 해먹은 레시피 히스토리 |
| 레시피 등록 | 사용자 기여 |
| 장보기 목록 | 부족한 재료 → 장보기 리스트 자동 생성 |
| 웹 버전 | Flutter Web 확장 |

## 추천 결과 카드 스펙

각 레시피 추천 카드에 표시할 정보:

```
┌─────────────────────────────────┐
│  [레시피 이미지]                 │
│                                  │
│  김치찌개                        │  ← 레시피 이름
│  ⏱ 20분  ·  🔥 쉬움             │  ← 조리시간 / 난이도
│                                  │
│  ██████████░░  80% 일치          │  ← 매칭률 프로그레스바
│                                  │
│  ⚠ 부족: 두부                    │  ← 부족한 재료
│                                  │
│  "보유한 김치와 돼지고기로        │  ← AI 추천 이유
│   간단하게 만들 수 있어요"        │
└─────────────────────────────────┘
```

| 요소 | 필수 | 출처 |
|------|------|------|
| 레시피 이름 | ✅ | DB |
| 대표 이미지 | ✅ | DB (없으면 기본 이미지) |
| 조리 시간 | ✅ | DB |
| 난이도 | ✅ | DB |
| 매칭률 | ✅ | RAG 파이프라인 ① 단계 |
| 부족한 재료 | ✅ | RAG 파이프라인 ① 단계 |
| AI 추천 이유 | ✅ | RAG 파이프라인 ④ LLM 응답 |

## 핵심 사용자 플로우

### 메인 시나리오: 재료 → 추천

```
앱 실행
  ↓
[홈 화면] — 냉장고 재료 현황 요약
  ↓
[재료 입력] — 검색 → 선택 → 수량/단위 → 추가
  ↓ (반복)
[냉장고 화면] — 등록된 재료 확인
  ↓
"레시피 추천받기" 버튼
  ↓
[로딩] — RAG 파이프라인 실행 (2~3초)
  ↓
[추천 결과] — 레시피 카드 3개
  ↓
카드 탭
  ↓
[레시피 상세] — 재료, 조리 순서, 팁
```

### 비회원 → 회원 전환

```
비회원으로 재료 등록 (로컬 저장)
  ↓
프로필 탭 → "로그인하면 데이터가 저장됩니다"
  ↓
소셜 로그인
  ↓
로컬 데이터 → Supabase 동기화
```

| 지표 | 목표 |
|------|------|

| 추천 소요 시간 | 3초 이내 |
| 일주일 재방문율 | 30% 이상 |
